# 象棋功能手动测试指南

本文档提供象棋功能的完整手动测试清单和步骤。

---

## 一、测试环境准备

### 1.1 启动服务

```bash
# 启动后端服务
npm start

# 或开发模式
npm run dev
```

### 1.2 访问测试页面

- 象棋广场：`http://localhost:3000/chess-plaza.html`
- 象棋房间：`http://localhost:3000/chess-room.html`

---

## 二、功能测试清单

### 2.1 房间管理

| 测试项 | 测试步骤 | 预期结果 |
|--------|----------|----------|
| 创建公开房间 | 在象棋广场点击"创建房间" | 房间创建成功，显示在房间列表 |
| 创建私密房间 | 勾选"私密"，设置密码 | 房间需要密码才能加入 |
| 加入公开房间 | 点击房间卡片 | 成功加入房间 |
| 加入私密房间 | 输入正确密码 | 成功加入房间 |
| 离开房间 | 点击"离开房间" | 正确离开，房间状态更新 |
| 删除房间 | 房主点击"解散房间" | 房间被删除，其他玩家被踢出 |

### 2.2 对战功能

| 测试项 | 测试步骤 | 预期结果 |
|--------|----------|----------|
| 游戏开始 | 双方玩家加入同一房间 | 游戏自动开始，显示棋盘 |
| 红方先行 | 观察开局 | 红方先行（默认） |
| 帅(将)移动 | 点击帅移动一格 | 只能在九宫内移动 |
| 仕(士)移动 | 点击仕移动 | 只能斜走，在九宫内 |
| 相(象)移动 | 点击相移动 | 走田字，不过河 |
| 車移动 | 点击車移动 | 直线行走，不能越子 |
| 馬移动 | 点击馬移动 | 日字形，蹩马腿检测 |
| 炮移动 | 点击炮移动 | 直线，吃子需隔一子 |
| 兵(卒)移动 | 点击兵移动 | 未过河直进，过河可左右 |
| 吃子 | 移动到对方棋子位置 | 正确吃掉对方棋子 |
| 将军提示 | 形成将军局面 | 显示"将军"提示 |
| 将帅对面 | 走出将帅对面 | 提示"将帅不能面对面" |
| 困毙检测 | 对方无合法走法 | 正确判定获胜方 |
| 认输 | 点击"认输"按钮 | 正确判定对方获胜 |
| 求和 | 点击"求和"按钮 | 发送求和请求给对方 |
| 接受求和 | 对方接受求和 | 游戏平局结束 |
| 拒绝求和 | 对方拒绝求和 | 游戏继续 |

### 2.3 AI对战

| 测试项 | 测试步骤 | 预期结果 |
|--------|----------|----------|
| AI代玩 | 玩家离开，启用AI | AI自动代玩家走棋 |
| AI难度 | 与AI对战 | AI能进行合理的走棋 |

### 2.4 观战功能

| 测试项 | 测试步骤 | 预期结果 |

### 2.5 积分系统

| 测试项 | 测试步骤 | 预期结果 |

## 六、测试用例示例

### 6.1 完整对局测试

```
1. 用户A创建公开房间
2. 用户B加入房间（黑方）
3. 游戏开始，红方先行
4. 红方走兵三进一
5. 黑方走卒3进1
6. 红方走马二进三
7. 黑方走马2进3
8. 红方走炮二平五
9. 黑方走炮8平5
10. 红方走马三进四（将军）
11. 黑方走将5平6（应将）
... 继续直到对局结束
```

### 6.2 将军测试

```
1. 摆出将军局面：红車在(0,6)，黑将在(0,0)
2. 红方走車六进一
3. 检查：黑方被将军
4. 黑方必须应将
```

### 6.3 困毙测试

```
1. 摆出困毙局面：黑方只剩将，红車在(0,4)
2. 红方走車四进一将军
3. 检查：黑方无合法走法
4. 预期：红方获胜
```

## 七、问题排查

### 7.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|----------|----------|
| 无法加入房间 | Token无效/房间已满 | 重新登录/检查房间状态 |
| 走棋无效 | 违反规则/不是轮到你 | 检查规则/等待对方走棋 |
| 棋盘不显示 | 前端加载错误 | 刷新页面/检查控制台 |
| Socket断连 | 网络不稳定 | 检查网络/重连机制 |
