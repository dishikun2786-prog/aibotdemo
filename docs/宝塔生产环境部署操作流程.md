# 宝塔线上生产环境部署操作流程

本文档为**操作流程清单**，便于按顺序执行。详细说明与故障排查见 [BT_PANEL_DEPLOYMENT.md](BT_PANEL_DEPLOYMENT.md)。

---

## 一、环境准备

| 步骤 | 操作 | 说明 |
|------|------|------|
| 1.1 | 登录宝塔面板 | 确保服务器已安装宝塔 |
| 1.2 | 安装软件 | 软件商店安装：**Node.js 版本管理器**（16.x/18.x）、**MySQL**（5.7/8.0）、**Nginx**；可选：**Redis**、**PM2**（或后续命令行安装） |
| 1.3 | 创建网站 | 网站 → 添加站点 → 填写域名 → 创建数据库（记下数据库名、用户名、密码） |

---

## 二、上传与安装项目

| 步骤 | 操作 | 说明 |
|------|------|------|
| 2.1 | 进入站点根目录 | 文件 → 选择站点目录（如 `/www/wwwroot/你的域名`） |
| 2.2 | 上传/拉取代码 | 上传项目压缩包并解压，或 SSH 执行：`git clone <仓库地址> .` |
| 2.3 | 安装依赖 | SSH 进入项目目录执行：`npm install --production` |
| 2.4 | 配置环境变量 | 复制 `cp .env.production.example .env`（若无则 `cp .env.example .env`），编辑 `.env` 填写下表项 |

**.env 必填项（生产）**

```env
# 与宝塔创建的数据库一致
MYSQL_HOST=localhost
MYSQL_PORT=3306
MYSQL_USER=数据库用户名
MYSQL_PASSWORD=数据库密码
MYSQL_DATABASE=数据库名

# 生产必须修改为强随机字符串（至少 32 位）
JWT_SECRET=请使用强随机字符串

# 站点域名（用于 CORS 与前端 API 地址）
CORS_ORIGIN=https://你的域名

# 服务端口（与 Nginx 反向代理一致）
PORT=3000
```

若使用 Redis、MongoDB、MiniMAX API，按 [CONFIG.md](CONFIG.md) 和 `.env.example` 补充。

---

## 三、数据库初始化与迁移

| 步骤 | 操作 | 说明 |
|------|------|------|
| 3.1 | 初始化基础库表 | 宝塔「数据库」→ phpMyAdmin → 选择该站点数据库 → 导入 `database/init_env.sql`。**注意**：若 `init_env.sql` 内为 `USE \`root\`;`，需改为你的数据库名，或先建好同名库再导入 |
| 3.2 | 执行迁移脚本 | 按顺序在 phpMyAdmin 中「导入」或命令行执行 `database/migrations/` 下尚未执行过的 SQL（如 `add_ai_agent_skins.sql`、`add_virtual_ai_agents.sql`、`add_story_system.sql` 等）。**迁移前请确认 SQL 内 `USE \`数据库名\`;` 与 `.env` 中 `MYSQL_DATABASE` 一致** |
| 3.3 | 创建管理员（若未有） | 在数据库中为 `users` 表插入管理员账号，或使用项目提供的创建管理员脚本（若有） |

---

## 四、静态资源与皮肤默认图

| 步骤 | 操作 | 说明 |
|------|------|------|
| 4.1 | 确认 public 目录 | 项目根目录下应有 `public` 文件夹，Node 会对外提供该静态目录 |
| 4.2 | 默认皮肤图 | 将 **180×320 像素的 PNG** 放到 `public/bg/180.png`，作为 AI 智能体对话默认背景；若暂无，可后台上传皮肤后再用 |

---

## 五、PM2 启动 Node 应用

| 步骤 | 操作 | 说明 |
|------|------|------|
| 5.1 | 安装 PM2 | `npm install -g pm2` |
| 5.2 | 启动应用 | 在项目根目录执行：`pm2 start server/app.js --name aibot` 或使用 `ecosystem.config.js`：`pm2 start ecosystem.config.js` |
| 5.3 | 开机自启 | `pm2 save` → `pm2 startup` 按提示执行 |
| 5.4 | 验证 | 执行 `curl http://127.0.0.1:3000/health` 应返回 `{"status":"ok",...}` |

---

## 六、Nginx 反向代理

| 步骤 | 操作 | 说明 |
|------|------|------|
| 6.1 | 编辑站点配置 | 网站 → 对应站点 → 设置 → 配置文件 |
| 6.2 | 配置反向代理 | 确保所有请求转发到 Node（端口与 `.env` 中 `PORT` 一致）。参考 [BT_PANEL_DEPLOYMENT.md#6](BT_PANEL_DEPLOYMENT.md) 的 Nginx 示例，需包含：`/`、`/api/`、`/socket.io/`、`/health`、静态资源（若需由 Nginx 直接提供可另配，本项目也可由 Node 的 `express.static('public')` 提供） |
| 6.3 | 保存并重载 | 保存后 Nginx 重载配置 |

---

## 七、前端访问地址配置

| 步骤 | 操作 | 说明 |
|------|------|------|
| 7.1 | 配置 API 与 Socket 地址 | 若前端写死 API 地址，改为生产域名（如 `https://你的域名`）；或通过管理后台「游戏配置」中的 `client_api_base`、`client_socket_url` 配置为 `https://你的域名`，确保与 Nginx 代理一致 |
| 7.2 | 访问验证 | 浏览器打开 `https://你的域名/game.html`、`https://你的域名/admin.html`，登录与接口、WebSocket 正常即可 |

---

## 八、SSL 与安全（推荐）

| 步骤 | 操作 | 说明 |
|------|------|------|
| 8.1 | 申请 SSL | 网站 → SSL → Let's Encrypt 申请并开启「强制 HTTPS」 |
| 8.2 | 权限与密钥 | `.env` 权限：`chmod 600 .env`；再次确认 `JWT_SECRET` 已改为强随机值 |

---

## 九、后续更新部署流程

1. 备份数据库（宝塔计划任务或 `node scripts/backup_database.js`）。
2. 拉取/上传新代码：`git pull` 或覆盖上传。
3. 执行：`npm install --production`。
4. 若有新增 SQL 迁移，在对应数据库执行 `database/migrations/` 下新脚本。
5. 重启应用：`pm2 restart aibot`（或你设置的进程名）。
6. 检查：`/health`、游戏页与后台功能。

## 十、AI 智能体皮肤相关检查（本次功能）

- 已执行 `database/migrations/add_ai_agent_skins.sql`（皮肤表、激活码表、用户拥有皮肤表、`users.current_skin_id`、默认皮肤数据）。
- 已存在 `public/bg/180.png`（180×320 PNG）或准备在后台上传皮肤后使用。
- 管理后台「AI智能体皮肤」Tab 可正常列表、新增/编辑、生成激活码、指定用户授权、导入。
- 游戏内 AI 对话「皮肤」入口可正常激活码激活、能量兑换、切换当前皮肤。
